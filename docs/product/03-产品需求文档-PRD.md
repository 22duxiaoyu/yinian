# Action 产品需求文档 PRD

> 版本：v1.1 复盘版
> 基线日期：2026-07-20
> 文档用途：描述当前产品范围、交互规则、验收标准和后续缺口，可作为继续开发的统一依据。

## 1. 背景与目标

Action 面向“记录很多但难以形成判断和行动”以及“目标明确但难以持续推进”的个人用户，通过结构化输入、AI 洞察、个人行动 Agent、人工确认、行动反馈和动态周报形成闭环。

本版本目标：

1. 让用户能在本地示例或云端个人空间中完成核心闭环；
2. 让 AI 输出具备依据、置信度和人工确认环节；
3. 让记录、文档、洞察、行动和周报在不同设备间同步；
4. 让 Agent 能澄清目标、生成计划和跟进反馈，同时不越过用户确认边界；
5. 保持界面以任务为主，不把产品做成通用聊天或复杂项目管理工具。

## 2. 范围

### 已纳入 v1.0

- 邮箱注册、验证、登录、退出和会话恢复；
- 本地完整示例和云端个人空间；
- 灵感、观察、复盘、行动四类快速输入；
- Word、PDF 文档导入与浏览器解析；
- 灵感库搜索、筛选、排序和堆叠卡片展开；
- 服务端 AI 洞察、证据引用、置信度与用户确认；
- 个人行动 Agent 的目标澄清、计划草案、人工确认、行动创建、阶段检查和重新规划；
- 洞察转行动、行动完成/恢复；
- 基于本周输入量动态选择布局的 AI 周报；
- 基于个人记录的引用式 AI 对话；
- Supabase 数据隔离、私有文件、实时同步；
- 桌面端、平板和移动端响应式适配。

### 暂不纳入

- 团队空间、成员邀请和协作权限；
- 日历、项目排期、依赖关系和复杂提醒；
- 自动执行外部任务；
- 公开分享笔记或报告；
- 原生 iOS、Android 客户端；
- 付费订阅、额度和账单系统；
- 企业级审计、组织知识库和完整运营权限系统。

## 3. 用户与权限

### 访客

- 可以进入“体验完整示例”；
- 示例数据仅保存在当前浏览器；
- 不支持跨设备同步。

### 云端用户

- 使用邮箱和密码注册、验证与登录；
- 只能访问自己的记录、文档、洞察、Agent 目标和周报；
- 支持跨设备同步与私有文档存储；
- AI 调用只读取当前用户的数据。

## 4. 功能需求

### FR-01 注册与登录

**需求：**登录和注册必须在同一区域通过顶部标签明确切换，避免用户同时面对两套操作。

规则：

- 注册要求邮箱、至少 6 位密码和二次确认；
- 注册成功后显示居中的成功反馈，明确是否需要邮箱验证；
- 用户可以返回登录或重新发送验证邮件；
- 邮箱未验证、密码错误、频率限制和网络失败应显示可理解的中文提示；
- 验证链接回到站点后自动恢复会话；
- 登录成功后加载当前用户的云端空间。

### FR-02 快速输入

**需求：**用户可以从顶部入口快速记录内容，不离开当前工作上下文。

字段：

- 类型：灵感、观察、复盘、行动；
- 标题：必填；
- 正文：必填；
- 标签：可选，多标签；
- 状态/情绪：可选；
- 行动类型可设置完成状态。

规则：

- 保存后显示清晰的成功动效；
- 云端用户自动同步，本地用户写入 LocalStorage；
- 标题和正文输入框保持轻微圆角，不使用过度胶囊化样式；
- 保存失败时不能清空用户输入。

### FR-03 文档导入

**需求：**用户可以导入 PDF 和 Word，系统将文件解析为可浏览内容。

规则：

- 单文件上限 20 MB；
- PDF 使用 pdf.js，DOCX 使用 Mammoth 在浏览器提取文本；旧版 DOC 无法提取正文时进入结构预解析；
- 解析结果包括摘要、关键词和最多若干内容卡片；
- 云端用户的原文件上传到私有存储桶，路径按用户 ID 隔离；
- 解析失败时保留文件信息并明确标记失败或降级状态；
- 文档可继续“沉淀为灵感”。

### FR-04 灵感库

**需求：**手动记录和导入文档进入同一列表，并从搜索区域下方连续排列。

规则：

- 卡片尺寸保持一致，按时间或用户选择的方式排序；
- 卡片按向下、向外的层叠关系排列；
- 点击卡片以抽卡方式展开，再次点击收回；
- 同一时刻只保留一张展开卡片；
- 支持全文关键词搜索、类型筛选、最新/最早/最长排序；
- 左侧灵感库数字实时等于当前有效记录和文档总数。

### FR-05 AI 洞察

**需求：**AI 基于最近 14 天的记录和文档，生成 3 条结构化洞察。

每条洞察至少包含：

- 标签、主题、标题和解释；
- 证据数量与证据引用 ID；
- 0-100 的置信度；
- 待确认、已确认或已驳回状态。

规则：

- 第一条为优先判断，后两条分别承载潜在张力和正向变化；
- “需要你确认”必须是页面最高权重区域；
- 生成过程显示全局处理状态，不让用户误以为按钮失效；
- 没有有效输入时返回空状态，不编造洞察；
- 用户可查看依据、确认或暂不采纳；
- 确认后可转成行动；
- 用户的确认状态写入云端，不能被页面刷新无故覆盖，并参与下一轮分析。

### FR-06 行动

**需求：**行动由用户新建或洞察转化，分为待开始和已完成。

规则：

- 卡片状态控件位于醒目但不过度突出的区域；
- 未完成时显示轮廓胶囊，完成后显示绿色胶囊；
- 完成动效结束后返回上一级列表；
- 完成与恢复未完成使用语义相反且视觉正确的反馈；
- 完成后引导用户选择“支持判断 / 结果不明确 / 不支持判断”并补充结果说明；
- 结果、完成时间和来源洞察写入云端，进入下一轮 AI 与周报；
- 支持查看详情、编辑、删除和恢复未完成；
- 完成率 = 已完成行动数 / 全部行动数；无行动时显示 0%。

### FR-06A 个人行动 Agent

**需求：**当用户直接提出一个持续目标时，Agent 需要先澄清目标和限制，再生成可确认的计划；用户确认后才能把步骤创建为行动。

规则：

- 新目标先进入 `clarifying`，Agent 至少确认结果、期限或限制条件中的必要信息；
- 信息足够后生成带顺序、预计时长、完成标准和可选截止时间的计划草案；
- 草案状态为 `ready` 时只能被确认或要求调整，不能提前创建行动；
- 用户确认后，计划步骤与行动建立一一对应关系，目标进入 `active`；
- 用户标记行动完成或恢复时，同步更新对应计划步骤；
- 阶段检查允许用户提交进展、阻碍和变化；如触发重新规划，新版本必须再次确认；
- 用户可以暂停、恢复或取消目标；取消只归档未完成的 Agent 行动，不删除历史记录；
- Agent 不能自行标记行动完成，也不能调用尚未接入的外部工具制造伪执行结果。

### FR-07 工作台

**需求：**工作台只呈现当前要处理的内容，不复制其他页面的完整信息。

优先级：

1. 今日推进：最近一项待处理行动；
2. 待确认洞察：引导用户查看依据和确认；
3. 最近输入：帮助用户接回上下文；
4. 其余数据只作为轻量状态，不做不可点击的装饰板块。

### FR-08 动态周报

**需求：**周报根据本周内容量和复杂度动态生成，不固定为同一版式和同一文案。

规则：

- 本周输入量 `<=3` 时优先精简布局 `sparse`；
- 输入量 `4-8` 时优先标准布局 `balanced`；
- 输入量 `>=9` 时优先深度布局 `dense`；
- AI 可根据主题复杂度修正布局，但只能返回上述三种值；
- 报告包含主题、摘要、主题标签、布局解释、行动反馈和下周实验；
- 下周实验包含标题、说明、持续天数、每日分钟数、观察指标和成功标准；
- 信息不足时明确写出证据不足；
- 当前触发方式为运行 AI 分析后生成/更新，不宣称每天自动后台生成。
- 保留多周历史快照，允许切换周期并对比输入、完成数和完成率趋势。

### FR-09 与 Action 对话

**需求：**AI 只能根据用户自己的记录、文档摘要和洞察回答。

规则：

- 回答结构为直接回答、引用依据和可选下一步；
- 必须区分原文事实、合理推断和仍需验证部分；
- 不替用户做最终决定；
- 对话入口显眼但不使用普通长输入框常驻页面；
- 无足够依据时明确说明，而不是使用通用知识补全。

### FR-10 云端同步与本地降级

**需求：**云端模式支持跨设备同步；未配置云端或云端失败时保留本地可用性。

规则：

- 首次登录可把本机数据迁移到云端；
- notes、documents、insights、weekly_reports 与 Agent 相关表通过 Realtime 刷新；
- 浏览器仍保留本地副本；
- 同步失败显示状态，不删除本地数据；
- 所有业务表启用 RLS，文件桶为私有桶。

### FR-11 数据控制、反馈与产品分析

**需求：**用户能控制自己的数据，产品团队能在不采集正文的前提下验证核心漏斗。

规则：

- 设置页支持完整 JSON 导出和恢复，覆盖资料、记录、文档、洞察、周报、事件与反馈；
- 用户输入“删除”二次确认后，可永久删除云端账户、业务数据和私有文件；
- 提供隐私说明、服务条款和 AI 可能出错的边界提示；
- 产品反馈与洞察确认反馈分表保存，不混入用户灵感语料；
- 行为埋点只记录必要结构化属性，不记录灵感正文；
- 提供按天聚合的核心漏斗视图，测试数据不对外宣称为业务成绩。

## 5. 非功能需求

### 安全

- DeepSeek API Key 只存放在 Supabase Secrets；
- 浏览器只使用 Supabase Publishable Key；
- 禁止把 service role key 写入前端、仓库或日志；
- 数据与文件都按 `auth.uid()` 隔离；
- AI 函数必须验证用户会话。

### 性能与反馈

- 点击后 100 ms 内提供视觉反馈；
- AI 长任务必须显示处理中状态；
- 数据列表更新不应造成页面整体跳动；
- 动效遵循 `prefers-reduced-motion`，减少动态时仍可完成操作。

### 响应式与可用性

- 桌面端不出现横向滚动；
- 移动端内容自然折行，只保留纵向滚动；
- 文本不遮挡按钮，按钮拥有明确可点击区域；
- 关键状态不只依赖颜色，还需文字或图标辅助。

## 6. 核心验收标准

### AC-01 完成首次闭环

**Given** 一个已验证的云端用户没有任何数据  
**When** 用户保存 3 条相关记录，运行 AI 洞察，确认一条并转为行动，再标记完成  
**Then** 灵感库、洞察、行动和周报都能看到相应更新，刷新页面后数据仍存在。

### AC-02 证据可追溯

**Given** AI 已生成一条洞察  
**When** 用户点击“查看依据”  
**Then** 系统只展示该用户的原始记录或文档，且至少能定位到洞察引用的来源。

### AC-03 跨设备同步

**Given** 同一用户在两台设备登录  
**When** 设备 A 新建或完成一项行动  
**Then** 设备 B 在实时通知或短暂刷新后看到相同状态。

### AC-04 AI 空状态

**Given** 用户没有记录或文档  
**When** 运行 AI 洞察  
**Then** 返回明确空状态，不产生虚构判断，原页面数据不丢失。

### AC-05 周报动态布局

**Given** 不同用户本周分别有少量、中量和大量输入  
**When** 生成周报  
**Then** 报告布局、主题数量和证据样本随内容变化，且所有结论能回到用户数据。

### AC-06 注册反馈

**Given** 用户提交有效邮箱和匹配密码  
**When** 注册成功  
**Then** 页面中心显示明确成功反馈、目标邮箱和下一步，不依赖页面底部的一行小字。

### AC-07 数据控制闭环

**Given** 云端用户已保存记录、文档、洞察、行动结果和历史周报  
**When** 用户导出完整数据或确认删除账户  
**Then** 导出文件可恢复上述对象；删除操作会移除账户、业务记录和私有文件，失败时不清除本机副本并明确提示。

### AC-08 Agent 人工确认边界

**Given** 用户创建了一个新的 Agent 目标
**When** 用户完成澄清并收到计划，但尚未点击确认
**Then** 行动页不存在该计划对应的任务；确认后才创建行动，完成行动会同步更新计划进度，重新规划仍需再次确认。

## 7. 当前已知缺口

| 缺口 | 风险 | 优先级 |
| --- | --- | --- |
| 主链路已埋点，但尚无真实用户规模的漏斗与留存数据 | 无法证明长期价值 | P0 |
| 已有基础 AI 回归集，但样本量和人工标注仍不足 | 不能宣称规模化准确率 | P0 |
| 真实邮箱验证和跨设备流程仍需持续人工回归 | 外部邮件与网络服务可能波动 | P0 |
| Agent 已完成闭环实现，但尚无长期真实目标样本 | 计划质量与重规划频率尚未被用户验证 | P0 |
| 文档解析主要在浏览器完成 | 大文件性能和格式兼容有限 | P1 |
| 自定义邮件已上线，但送达率尚未形成持续监控 | 国内外邮箱可能存在延迟或拦截 | P1 |
| 两设备并发编辑尚无冲突合并策略 | 可能出现后写覆盖 | P1 |
| 扫描 PDF 和复杂表格尚无 OCR/服务端解析 | 部分导入内容无法完整提取 | P1 |

## 8. 下一阶段建议

1. 招募 5-10 名目标用户完成首次任务测试，并连续观察 4 周；
2. 将现有 AI 回归集扩展到至少 50 组匿名人工标注样本；
3. 增加真实邮箱注册、验证和两设备同步的定期回归；
4. 验证扫描 PDF、复杂表格和超长文档，再决定是否迁移到服务端解析；
5. 用 5-10 个真实目标验证 Agent 的澄清质量、计划确认率、首步完成率和重新规划价值；
6. 根据真实漏斗和访谈判断是否需要提醒、定时周报或团队能力。
