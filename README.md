# Action · AI 思考工作台

Action 是一款把零散想法和明确目标转化为有依据判断与可执行计划的个人行动 Agent。产品围绕“捕捉 → 理解 → 计划 → 行动 → 反馈”建立完整闭环，而不是停留在记录和搜索层面。

在线体验：https://www.actionmind.cn/

管理员数据中心：https://www.actionmind.cn/admin.html

## 产品能力

- 快速输入灵感、观察、复盘和行动
- 从近期输入中识别重复主题与潜在张力
- 为每条洞察展示置信度和原始证据
- 用户可确认或驳回 AI 假设，反馈写回云端并参与下一轮分析
- 将洞察转成行动，记录支持/不明确/不支持等真实结果
- 个人行动 Agent 主动澄清目标、生成分步计划，并在用户确认后创建行动
- 根据完成结果和阶段反馈发起检查、调整计划，所有改动再次交由用户确认
- 生成动态周报，保留多周历史快照和周期趋势
- 全局搜索、详情、编辑、删除、Word/PDF 导入与完整 JSON 备份恢复
- 本地模式与 Supabase 云端账户双模式
- 跨设备实时同步、私有文档存储与本地数据首次迁移
- 服务端 AI 洞察、引用式对话、行动 Agent 与动态周报
- 主链路埋点、产品反馈、账户永久删除与隐私说明
- 管理员数据中心，覆盖活跃、转化漏斗、功能使用、AI 质量、用户激活和反馈
- 桌面与移动端 Playwright 主路径回归、基础 AI 评测集
- 桌面端、平板和移动端响应式界面

## AI 产品设计

未配置云端时，Action 使用本地规则引擎验证人机协作工作流；配置 Supabase 与服务端 DeepSeek 后，洞察、引用式对话、行动 Agent 和动态周报由服务端 AI 生成，模型密钥不会进入浏览器。Agent 在计划被用户确认前不会创建任务，也不会自行标记完成。

产品围绕以下指标建立评测：

- 北极星指标：每周被确认并进入行动的洞察数
- 质量指标：洞察确认率、证据打开率、洞察驳回率
- 护栏指标：错误引用率、行动撤销率、主动建议关闭率

## 本地运行

```bash
python3 -m http.server 6002
```

打开 `http://localhost:6002`，选择“体验完整示例”即可进入预置作品集空间。

## 验证

```bash
npm test
npm run eval
npm run test:e2e
```

静态测试检查页面契约、云端能力和隐私删除链路；离线评测覆盖基础 AI 安全/质量场景；Playwright 在桌面与移动端验证“输入 → 洞察 → 行动结果 → 周报”以及“目标 → 澄清 → 确认计划 → 行动”的主路径。

## 数据说明

未填写 `cloud-config.js` 时，数据只保存在浏览器 LocalStorage 中。配置云端后，邮箱账号、记录、文档、洞察、Agent 目标、对话、计划、检查记录、行动结果、行为事件和周报会按用户隔离保存在 Supabase；本机仍保留一份副本，并支持跨设备实时刷新。设置页支持数据导出、恢复和永久删除账户。

云端部署步骤见 [CLOUD_SETUP.md](./CLOUD_SETUP.md)。生产环境已配置自定义邮件域名和 SMTP，注册邮件发送六位数字验证码，验证码在当前注册页直接完成校验。
