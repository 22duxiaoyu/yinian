# Action · AI 思考工作台

Action 是一款把零散想法提炼成洞察与行动的 AI 思考工作台。产品围绕“捕捉 → 理解 → 行动 → 反馈”建立完整闭环，而不是停留在记录和搜索层面。

在线体验：https://22duxiaoyu.github.io/yinian/

## 产品能力

- 快速输入灵感、观察、复盘和行动
- 从近期输入中识别重复主题与潜在张力
- 为每条洞察展示置信度和原始证据
- 用户可确认或驳回 AI 假设，保留最终决策权
- 将 AI 建议一键转成行动，并用完成结果更新转化率
- 自动生成本周思考简报
- 全局搜索、详情查看、编辑、删除、导入和导出
- 本地模式与 Supabase 云端账户双模式
- 跨设备实时同步、私有文档存储与本地数据首次迁移
- 服务端 AI 洞察、引用式对话与动态周报
- 桌面端、平板和移动端响应式界面

## AI 产品设计

未配置云端时，Action 使用本地规则引擎验证人机协作工作流；配置 Supabase 与服务端 DeepSeek 后，洞察、引用式对话和动态周报由服务端 AI 生成，模型密钥不会进入浏览器。

产品围绕以下指标建立评测：

- 北极星指标：洞察进入行动的周转化率
- 质量指标：洞察确认率、证据打开率、洞察驳回率
- 护栏指标：错误引用率、行动撤销率、主动建议关闭率

## 本地运行

```bash
python3 -m http.server 6002
```

打开 `http://localhost:6002`，选择“体验完整示例”即可进入预置作品集空间。

## 数据说明

未填写 `cloud-config.js` 时，数据只保存在浏览器 LocalStorage 中。配置云端后，邮箱账号、记录、文档、洞察和周报会按用户隔离保存在 Supabase；本机仍保留一份副本，并支持跨设备实时刷新。

云端部署步骤见 [CLOUD_SETUP.md](./CLOUD_SETUP.md)。
