隐私说明

Action 处理的是你的个人记录，因此默认原则很简单：数据归你、用途透明、删除有效。

1. 两种使用方式

体验示例和本地模式的数据保存在当前浏览器中，不会自动上传。登录云端账户后，资料、灵感、导入文档、洞察、行动结果和周报会同步到你的私有云端空间，便于跨设备使用。

2. AI 会接触什么

只有当你主动点击“用 AI 做洞察”、生成周报或向 Action 提问时，相关记录与文档解析内容才会被发送给当前配置的 DeepSeek 模型服务生成结果。AI 可能出错；系统不会替你确认洞察，也不会修改原始记录。

3. 使用的服务

Action 使用 Supabase 提供登录、数据库、私有文件存储和云函数能力，使用 DeepSeek 提供模型推理，并通过 GitHub Pages 提供前端页面。各服务对传输数据的处理同时受其公开政策约束。

4. 安全与隔离

云端业务表启用了行级权限控制，登录用户只能访问自己的记录；导入文件和头像存储在私有空间中。前端不会保存模型服务密钥。

5. 导出与删除

你可以在“个人设置 → 数据管理”导出完整 JSON 备份，也可以恢复备份。在“隐私与账户”中确认删除后，账户、数据库记录和私有文件会一并永久删除，此操作无法撤销。

6. 产品分析

Action 会记录必要的产品事件，例如完成登录、保存输入、查看证据、确认洞察和回填行动结果，用于判断流程是否有效。事件属性限制为非正文的结构化字段，不把灵感正文当作行为分析属性。

7. 联系与更新

项目仍处于个人作品集验证阶段。问题可通过 GitHub 项目页反馈。规则变化时，本页日期会同步更新。
